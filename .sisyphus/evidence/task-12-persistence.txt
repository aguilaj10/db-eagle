===================================================

## Overview
Implemented encrypted connection profile persistence using Java Preferences and AES-GCM encryption.

## Implementation Details

### Storage Format:
- Java Preferences node: "com.dbeagle.profiles"
- Each profile stored as JSON with key = profile.id
- Profile JSON contains:
  - All non-sensitive fields in plaintext
  - encryptedPassword field as EncryptedData (ciphertext, iv, salt)
  
### Encryption:
- Uses existing CredentialEncryption.kt with AES-GCM
- Master password obtained via MasterPasswordProvider callback
- PBKDF2 key derivation with 100,000 iterations
- 32-byte salt, 12-byte IV per encrypted field

### Master Password Provider:
- Implemented as fun interface (SAM)
- Suspend function for async UI integration
- Allows UI layer to implement dialog/prompt separately

### DatabaseType Handling:
- DatabaseType is sealed class (PostgreSQL, SQLite objects)
- Cannot use .name or valueOf() directly
- Implemented manual discriminator mapping to/from String
