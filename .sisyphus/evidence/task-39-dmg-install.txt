Enhanced the `macOS { }` block in `compose.desktop.application.nativeDistributions` with:

1. **Bundle Identifier**: `bundleID = "com.dbeagle.app"`
   - Required for macOS packaging
   - Uses reverse-DNS style: com.dbeagle.app

2. **DMG Customization**: 
   - `dockName = "DBEagle"` (name shown in macOS dock/applications folder)

3. **Optional Code Signing**:

     - `DBEAGLE_MAC_SIGN_IDENTITY` (or `dbeagle.mac.sign.identity`)
     - `DBEAGLE_MAC_KEYCHAIN` (or `dbeagle.mac.keychain`) [optional]

   - Signing block only activated when credentials present

4. **Optional Notarization**:
   - Requires signing to be enabled + Apple credentials:
     - `DBEAGLE_APPLE_ID` (or `dbeagle.apple.id`)
     - `DBEAGLE_APPLE_TEAM_ID` (or `dbeagle.apple.team.id`)
     - `DBEAGLE_APPLE_APP_PASSWORD` (or `dbeagle.apple.app.password`)
   - Notarization block only activated when all credentials present
   - If not provided, DMG created without notarization (still installable with Gatekeeper warning)

## Drag-to-Applications Install Flow

**Implementation Status**: Default Compose Desktop behavior

Compose Desktop's `packageDmg` task uses Apple's `jpackage` tool which automatically:
1. Creates a DMG volume with the application bundle
2. Includes a symbolic link to `/Applications` folder in the DMG
3. User experience:
   - Mount DMG (double-click)
   - Drag `DBEagle.app` to the Applications folder link
   - Eject DMG
   - Launch from Launchpad/Applications folder

**Note**: Custom DMG background images require additional jpackage configuration not implemented in this task. The default layout includes the app icon and Applications symlink side-by-side.

## Code Signing & Notarization Configuration

### Unsigned Build (Default)
No environment variables set → DMG created unsigned → Works on local machine with "Open Anyway" in System Preferences.

### Signed Build (Optional)
Set `DBEAGLE_MAC_SIGN_IDENTITY` → DMG signed with Developer ID → Passes Gatekeeper on user machines.

### Signed + Notarized Build (Optional)
Set signing vars + Apple ID vars → DMG signed + notarized → No warnings on any Mac.

**Example Environment Variables (DO NOT COMMIT TO VERSION CONTROL)**:
```bash
export DBEAGLE_MAC_SIGN_IDENTITY="Developer ID Application: Your Name (TEAM123)"
export DBEAGLE_APPLE_ID="your.email@example.com"
export DBEAGLE_APPLE_TEAM_ID="TEAM123"
export DBEAGLE_APPLE_APP_PASSWORD="xxxx-xxxx-xxxx-xxxx"  # App-specific password
```

**Alternative: gradle.properties (local only, git-ignored)**:
```properties
dbeagle.mac.sign.identity=Developer ID Application: Your Name (TEAM123)
dbeagle.apple.id=your.email@example.com
dbeagle.apple.team.id=TEAM123
dbeagle.apple.app.password=xxxx-xxxx-xxxx-xxxx
```
