Behavior verified (high level):
- Multiple active connections can exist simultaneously (3 sessions opened in test).
- UI uses a dedicated connection-tab row (separate from main navigation tabs) and switching connection tabs swaps the active profile context.
- Per-connection isolation is implemented by keying state by profileId (SQL editor text, last executed SQL/result, schema browser cache/state).
- Closing a connection tab triggers both:
  - driver.disconnect()
  - DatabaseConnectionPool.closePool(profileId) (via injected callback for testability)
