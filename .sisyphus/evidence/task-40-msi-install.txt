**Cannot execute actual MSI packaging on Linux:**
- MSI format requires Windows-specific WiX Toolset (`candle.exe`, `light.exe`)
- Compose Desktop `packageMsi` task delegates to `jpackage` with `--type msi`, which requires Windows
- Current environment: Linux x86_64 (headless server)

**Verification approach:**
- Dry-run confirms task graph is valid

## CI/CD Recommendations

### GitHub Actions Windows Runner
```yaml
jobs:
  windows-msi:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: 'temurin'
          java-version: '17'
      - name: Build MSI
        run: ./gradlew :app:packageMsi
      - name: Upload MSI
        uses: actions/upload-artifact@v4
        with:
          name: DBEagle-Windows-MSI
          path: app/build/compose/binaries/main/msi/*.msi
```

### Optional Code Signing (Post-Processing)
```yaml
      - name: Sign MSI (if certificate available)
        if: secrets.WIN_CERT_BASE64 != ''
        run: |
          echo "${{ secrets.WIN_CERT_BASE64 }}" | base64 --decode > cert.pfx
          signtool.exe sign /f cert.pfx /p "${{ secrets.WIN_CERT_PASSWORD }}" /tr http://timestamp.digicert.com /td sha256 /fd sha256 app/build/compose/binaries/main/msi/DBEagle-1.0.0.msi
          rm cert.pfx
```

## Start Menu & PATH Behavior

### Start Menu Integration
- MSI creates shortcuts in `%APPDATA%\Microsoft\Windows\Start Menu\Programs\DBEagle\`
- Shortcut launches `DBEagle.exe` from install directory
- Uninstall removes shortcuts automatically

### Add to PATH (Not Implemented)
- Plan mentioned "Add to Windows PATH (optional)" as stretch goal
- Compose Desktop DSL does not provide declarative PATH modification
- Implementation would require:
  1. Custom WiX XML snippet via `jpackage --win-dir-chooser`
  2. Or post-install script modifying registry `HKCU\Environment\Path`
- Decision: Out of scope for basic MSI packaging (can be added later if user demand exists)
